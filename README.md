# Why ScriptableObjects?

ScriptableObjects separate game data from game logic, making Unity projects easier to organize, maintain, and expand.

## Benefits

- Clean architecture
- Reusable data assets
- Better project organization
- Easier balancing
- Inspector-friendly workflow
- Reduced hardcoded values
- Improved scalability

# Best Practices

- Keep game data inside ScriptableObjects.
- Separate gameplay logic from configuration.
- Create reusable assets instead of duplicating data.
- Use meaningful naming conventions.
- Organize ScriptableObjects into dedicated folders.
- Avoid storing runtime state inside ScriptableObjects.
